## Vercel Firewall Incident

The live site was returning `403 Forbidden` for API routes even though static pages loaded.

Observed responses:

- `https://promptlab.tools/mobile/` returned `200`.
- `https://promptlab.tools/app/` returned `200`.
- `OPTIONS https://promptlab.tools/api/proxy` returned `403`.
- `POST https://promptlab.tools/api/proxy` returned `403`.
- `POST https://promptlab.tools/api/telemetry` returned `403`.

The 403 responses had:

- `X-Vercel-Mitigated: deny`

Root cause was an enabled Vercel Firewall custom rule:

- Rule: `Emergency block all Prompt Lab API compute`
- ID: `rule_emergency_block_all_prompt_lab_api_compute_CrQDH4`
- Action: `Deny`
- Status before fix: `Enabled`
- Condition: `path starts with /api/`
- Description: `Cost containment block for all /api routes while billing incident is investigated`

Action taken:

- Disabled only `Emergency block all Prompt Lab API compute`.
- Published the firewall config to production with `npx vercel firewall publish --yes`.
- Left `Emergency block billing license route` enabled.

Remaining enabled firewall rule:

- Rule: `Emergency block billing license route`
- ID: `rule_emergency_block_billing_license_route_Sx8pi7`
- Action: `Deny`
- Status: `Enabled`
- Condition: `path equals /api/billing/license`

Post-fix verification:

- `OPTIONS https://promptlab.tools/api/proxy` returned `204`.
- `POST https://promptlab.tools/api/proxy` returned `200`.
- Live proxy response included:
  - `X-Demo-Remaining: 9`
  - `X-Hosted-Provider: anthropic`
  - `X-Hosted-Model: claude-sonnet-4-20250514`
  - `X-Ratelimit-Store: memory`
- Browser smoke against `https://promptlab.tools/mobile/` passed live mobile generation.
